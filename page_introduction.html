Introduction

wxAutoExcel is a C++ library which aims to make automating Microsoft Excel less painful.

It requires wxWidgets 3.1 or newer and works only on Microsft Windows. While it does not comes even near to covering all the classes Microsoft Excel exposes, it still supports their considerable number. You can see what classes are supported here. Be aware that even if a class is in the lists, some of its methods or properties may not still be supported. wxAutoExcel does not support any Microsot Excel events.

It is assumed that the users of wxAutoExcel are familar with VBA and Microsoft Excel model, as the library is only a C++ wrapper around it. But even if they are not, it could still possible to use wxAutoExcel, there are numerous resources regarding Microsoft Excel VBA programming on the internet. Moreover, very often it is possible to just record the macro inside the Microsoft Excel and translate the resulting VBA code into corresponding wxAutoExcel calls, which is actually how I write most of my code using wxAutoExcel.
